  <files>scripts/generate-postal-codes.ts, scripts/generate-postal-codes.test.ts</files>
  <behavior>

    - Test 2: parseXlsx extracts postal_code (5 chars), postal_area_name, postal_area_name_sv, municipality code (3-digit padded) from rows starting at row 4
    - Test 3: fetchSwedishMunicipalityNames returns a Map of 3-digit padded code to Swedish name
    - Test 4: Municipality codes are padded to 3 digits with leading zeros (e.g., 91 becomes "091")
    - Test 5: Entries with missing Swedish municipality name throw an error (data integrity check)
    - Test 6: Output JSON structure matches PostalCodeEntry interface (all 5 required fields present, no extra fields)
  </behavior>
  <action>
Create `scripts/generate-postal-codes.ts` following the research patterns exactly. The script has 4 functions and a main entrypoint:

**1. `downloadXlsx(): Promise<{ buffer: ArrayBuffer; year: number }>`**
- Try current year URL: `https://stat.fi/media/uploads/tup/paavo/alueryhmittely_posnro_${year}_fi.xlsx`
- If response is not OK, try `year - 1`
- If both fail, throw error
- Log which year was used
- Return the ArrayBuffer and the year used (DATA-03)

**2. `fetchSwedishMunicipalityNames(year: number): Promise<Map<string, string>>`**
- Fetch from: `https://api.stat.fi/classificationservice/open/api/classifications/v2/classifications/kunta_1_${year}0101/classificationItems?content=data&meta=max&lang=sv&format=json`
- Use the SAME year as the XLSX download (prevents municipality code mismatches)
- Parse response JSON into `Map<municipalityCode, swedishName>`
- Pad municipality code to 3 digits with `.padStart(3, '0')` (DATA-04)
- Throw error if fetch fails

**3. `parseXlsx(buffer: ArrayBuffer, svNames: Map<string, string>): Promise<PostalCodeEntry[]>`**
- Use `new ExcelJS.Workbook()` and `workbook.xlsx.load(buffer)` — note: the method is `.xlsx.load()` NOT `.xlsx.loadFromBuffer()`. ExcelJS `load()` accepts `Buffer | ArrayBuffer`.
- Access first worksheet by index: `workbook.worksheets[0]` (NEVER by name — name changes between years) (DATA-01)
- Skip rows 1-3 (title, note, headers). Data starts at row 4.
- For each data row, extract:
  - Column 1: `postal_code` (trim, skip if empty or not 5 chars)
  - Column 2: `postal_area_name` (Finnish area name)
  - Column 3: `postal_area_name_sv` (Swedish area name)
  - Column 4: municipality code (convert to string, padStart 3 with '0')
  - Column 5: `municipality_name` (Finnish municipality name) (DATA-02)
- Look up Swedish municipality name from `svNames` map using the padded municipality code
- Throw error if any municipality code has no Swedish name match (data integrity)
- Return array of PostalCodeEntry objects

**4. `main()`**
- Call downloadXlsx, fetchSwedishMunicipalityNames (with XLSX year), parseXlsx
- Log entry count
- Create output directory: `public/data/`
- Write JSON to `public/data/postal-codes.json` with `JSON.stringify(entries, null, 2)`
- Log output path

**TypeScript interfaces:**
```typescript
interface PostalCodeEntry {
  postal_code: string           // "00100"
  postal_area_name: string      // "Helsinki keskusta - Etu-Töölö"
  postal_area_name_sv: string   // "Helsingfors centrum - Främre Tölö"
  municipality_name: string     // "Helsinki"
  municipality_name_sv: string  // "Helsingfors"
}

interface ClassificationItem {
  code: string
  classificationItemNames: Array<{ lang: string; langName: string; name: string }>
}
```

**IMPORTANT — ExcelJS load method:** Use `workbook.xlsx.load(buffer)` where `buffer` is the `ArrayBuffer` returned by `response.arrayBuffer()`. The ExcelJS `.load()` method accepts both `Buffer` and `ArrayBuffer`. Do NOT use `.loadFromBuffer()` (does not exist in ExcelJS API).

Also create `scripts/generate-postal-codes.test.ts` with unit tests for the parsing and municipality code padding logic. Use Node's built-in `node:test` and `node:assert` (no need for Jest — the project doesn't have it yet). Tests should verify:
- Municipality code padding (number 91 becomes "091")
- PostalCodeEntry shape validation
- Error thrown when Swedish name missing for a municipality code
