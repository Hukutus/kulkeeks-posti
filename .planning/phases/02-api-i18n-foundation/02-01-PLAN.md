Create the Posti delivery API proxy Route Handler and typed library wrapper.

Output: Two files — a reusable typed lib (`get-delivery-dates.ts`) for Server Components (Phase 3+) and a Route Handler (`route.ts`) for client-side fetches.

       - `export const dynamic = 'force-dynamic'` at module scope (prevents Next.js caching)
       - `export async function GET(request: NextRequest)` that:
         a. Reads `postalCode` from `request.nextUrl.searchParams`
         b. Validates format: must be exactly 5 digits (`/^\d{5}$/`), else return 400 `{ error: "Invalid postal code. Must be a 5-digit number." }`
         c. Calls Posti API server-to-server with `cache: 'no-store'`
         d. On non-200 from Posti: return 502 `{ error: "Posti API returned status {N}" }`
         e. Validates response body with `DeliverySchema.safeParse(raw)` (imported from `@/lib/get-delivery-dates`)
         f. On Zod failure: return 502 `{ error: "Unexpected response shape from Posti API" }`
         g. On success: return `NextResponse.json(parsed.data)`
         h. On catch (network error): return 502 `{ error: "Posti API is unavailable" }`
       - Follow the exact pattern from 02-RESEARCH.md Pattern 1
