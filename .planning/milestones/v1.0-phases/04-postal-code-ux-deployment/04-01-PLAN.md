**1. Create `src/app/api/geocode/route.ts`:**

Create a GET Route Handler that proxies reverse geocoding to Digitransit:
- Accept `lat` and `lon` query parameters (both required, return 400 if missing)
- Read `process.env.DIGITRANSIT_API_KEY` (server-only, no NEXT_PUBLIC_ prefix)
- Fetch `https://api.digitransit.fi/geocoding/v1/reverse?point.lat=${lat}&point.lon=${lon}&size=1` with header `digitransit-subscription-key: ${apiKey}` (only include header if key exists -- allows fallback testing without key)
- Extract `data.features[0].properties.postalcode` from response
- If no postal code found, return `{ error: "No postal code found for location" }` with status 404
- On fetch error, return `{ error: "Geocoding failed" }` with status 502
- On success, return `{ postalCode: string }`
- Set `export const dynamic = 'force-dynamic'` (geolocation is per-request)

**2. Create `src/components/PostalCodeGate.tsx`:**

Create a `'use client'` component that orchestrates postal code state:
- Use `useState` for `postalCode: string | null` (initially null) and `status: 'loading' | 'resolved' | 'selecting'` (initially 'loading')
- `STORAGE_KEY = 'posti-days:postalCode'`
- `useEffect` on mount: read `localStorage.getItem(STORAGE_KEY)`. If found, set postalCode and status='resolved'. If not, set status='selecting'.
- `handleCodeSelected(code: string)`: write to localStorage, set postalCode, set status='resolved'
- `handleChangeCode()`: remove from localStorage, set postalCode=null, set status='selecting'
- Render logic:
  - `loading`: show a minimal loading indicator (a subtle pulse dot or the dialect question text as placeholder) -- keep it lightweight, avoid layout shifts
  - `selecting`: render `<PostalCodeSelector onCodeSelected={handleCodeSelected} />`
  - `resolved`: render `<DeliveryDisplay postalCode={postalCode!} onChangeCode={handleChangeCode} />`

**3. Create `src/components/PostalCodeSelector.tsx`:**

Create a `'use client'` component that handles geolocation flow:
- Props: `{ onCodeSelected: (code: string) => void }`
- State: `geoState: 'requesting' | 'denied' | 'error'` (initially 'requesting')
- `useEffect` on mount: call `navigator.geolocation.getCurrentPosition(successCallback, errorCallback, { timeout: 10000 })`
  - Success: fetch `/api/geocode?lat=${latitude}&lon=${longitude}`, if OK call `onCodeSelected(postalCode)`, else set geoState='error'
  - Error: set geoState='denied'
- When geoState is 'requesting': show a centered message asking user to allow location access (use translations from 'PostalCode' namespace: `allowLocation` key). Show a subtle spinner/pulse animation.
- When geoState is 'denied' or 'error': show a message that location was denied/failed and display a placeholder text input with a note "Autocomplete coming in next update" (Plan 04-02 will replace this with the real Combobox). Include a "Try again" button that re-triggers geolocation.
- IMPORTANT: This is a stub for geolocation-only flow. Plan 04-02 will add the full autocomplete. The stub should show the geoState status and have a basic text input that accepts a 5-digit postal code manually (for testing). On submit of a valid 5-digit code, call `onCodeSelected(code)`.

Style with existing Tailwind classes matching the app's stone/warm aesthetic (bg-stone-50 dark:bg-stone-950, stone colors, rounded-2xl cards).
