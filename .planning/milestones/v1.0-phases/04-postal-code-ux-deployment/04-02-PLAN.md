**2. Rewrite `src/components/PostalCodeSelector.tsx`:**

Replace the stub manual input from Plan 04-01 with a full autocomplete implementation. Keep the existing geolocation flow (requesting/denied/error states) intact.

**Postal codes data loading:**
- Do NOT statically import postal-codes.json (would inflate bundle by ~250KB)
- Instead, fetch from `/data/postal-codes.json` on first autocomplete open (the file is in public/data/)
- Cache the loaded data and Fuse instance in module-level variables (`let cachedData: PostalEntry[] | null = null`, `let fuseInstance: Fuse<PostalEntry> | null = null`)
- Create an async `loadPostalCodes()` function that fetches + initializes Fuse if not cached

**Fuse.js configuration:**
```typescript
const fuse = new Fuse(data, {
  keys: [
    { name: 'postal_code', weight: 2 },
    { name: 'postal_area_name', weight: 1 },
    { name: 'postal_area_name_sv', weight: 1 },
    { name: 'municipality_name', weight: 1 },
    { name: 'municipality_name_sv', weight: 0.5 },
  ],
  threshold: 0.3,
  minMatchCharLength: 2,
})
```
- Search with `fuse.search(query)` limited to 50 results via `.slice(0, 50)` after search

**Grouping by municipality:**
- Create a `groupByMunicipality(entries: PostalEntry[])` function that returns `Map<string, PostalEntry[]>`
- Iterate entries, group by `municipality_name`

**Combobox component (Headless UI v2):**
- Import `{ Combobox, ComboboxInput, ComboboxOptions, ComboboxOption }` from `@headlessui/react`
- State: `query: string`, `results: PostalEntry[]`, `postalCodes: PostalEntry[] | null` (null until loaded)
- On ComboboxInput focus (first time): call `loadPostalCodes()` and set postalCodes state
- On ComboboxInput onChange: if query.length >= 2, search with Fuse, set results. If < 2, clear results.
- ComboboxOptions rendering:
  - Group results with `groupByMunicipality(results)`
  - For each municipality group, render:
    - A non-interactive `<li>` heading with municipality name (text-xs font-semibold text-stone-500 uppercase tracking-wide px-3 py-1)
    - For each postal entry in the group, render a `<ComboboxOption value={entry}>` with:
      - Postal code in mono font (font-mono text-stone-400 mr-2)
      - Area name in regular font
      - `focus` render prop for bg-stone-100 dark:bg-stone-800 highlight
- Combobox onChange: when entry selected, call `onCodeSelected(entry.postal_code)`
- Style the ComboboxInput: `w-full border border-stone-300 dark:border-stone-700 rounded-xl px-4 py-3 bg-white dark:bg-stone-900 text-stone-800 dark:text-stone-200 placeholder-stone-400 focus:outline-none focus:ring-2 focus:ring-stone-400`
- Style ComboboxOptions: `mt-1 bg-white dark:bg-stone-900 shadow-lg rounded-xl max-h-64 overflow-auto border border-stone-200 dark:border-stone-800`
- Use the `anchor="bottom"` prop on ComboboxOptions for positioning
- Placeholder text from translations: `t('searchPlaceholder')`

**Geolocation states rendering (keep from 04-01 but integrate better):**
- 'requesting': Show centered message with subtle spinner + "Allow location access" text. Below, show a muted link "Or search manually" that sets geoState to 'denied' to skip waiting.
- 'denied' / 'error': Show the Combobox autocomplete as the primary UI. Above it, show a small "Try location again" button. Show the error message in subtle text above the combobox.

**Integration with PostalCodeGate:**
- Keep the same props interface: `{ onCodeSelected: (code: string) => void }`
- Use `useTranslations('PostalCode')` for all user-facing strings

**Accessibility:**
- Headless UI handles keyboard nav (arrow keys, Enter, Escape) and aria attributes automatically
- Add `aria-label` on the ComboboxInput for screen readers
