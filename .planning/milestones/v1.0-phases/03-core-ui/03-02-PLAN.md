Replace the contents of `src/app/[locale]/page.tsx` with a full async Server Component that:

**1. Imports:**
- `connection` from `next/server`
- `getTranslations`, `getFormatter`, `setRequestLocale` from `next-intl/server`
- `getDeliveryDates` from `@/lib/get-delivery-dates`
- `getTodayISO`, `getCurrentWeekISO`, `filterWeekDeliveries`, `isDeliveryDay` from `@/lib/delivery-utils`
- `content` from `../../../content.json`

**2. Component setup:**
- Type: `{ params: Promise<{ locale: string }> }`
- `const { locale } = await params`
- `setRequestLocale(locale)` — required for next-intl static generation
- `await connection()` — MUST come before Math.random() to force per-request rendering
- Select random dialect: `content.dialects[Math.floor(Math.random() * content.dialects.length)]`
- Fetch delivery data: `const result = await getDeliveryDates('00100')` with `// TODO Phase 4: replace with resolved postal code` comment
- `const t = await getTranslations('Delivery')`
- `const format = await getFormatter()`

**3. Error state:**
If `!result.success`, render an error card:
- Centered container with `bg-stone-50 dark:bg-stone-950` background
- Error icon or text showing `t('errorTitle')`
- Display `result.error` in muted text
- Still show the dialect question in Caveat font (the dialect is independent of delivery data)

**4. Success state — Main layout:**
Use a mobile-first centered layout:
- `<main className="min-h-screen bg-stone-50 dark:bg-stone-950 flex flex-col items-center px-4 py-12 sm:py-16">`
- Inner container: `max-w-sm w-full mx-auto text-center`

**5. Dialect question/answer block (DISP-01, DISP-02, DISP-03, VIS-01):**
- Dialect question: `<p className="font-handwriting text-3xl sm:text-4xl text-stone-700 dark:text-stone-300 leading-relaxed">{dialect.question}</p>`
- Answer: `<p className="font-handwriting text-7xl sm:text-8xl font-bold mt-4 {color}">{deliveryToday ? dialect.yes : dialect.no}</p>`
- Color: `deliveryToday ? 'text-green-600 dark:text-green-400' : 'text-red-600 dark:text-red-400'`
- The answer MUST be the largest element on the page — it's the primary information

**6. Dialect metadata (DISP-04):**
- Below the answer: `<p className="text-stone-500 dark:text-stone-400 text-sm mt-3 italic">{dialect.dialect} — {dialect.region}</p>`

**7. Week view (DISP-05):**
- Section title: `<h2 className="text-lg font-semibold text-stone-700 dark:text-stone-300 mb-3">{t('weekTitle')}</h2>`
- Wrapped in a card: `<div className="mt-10 w-full bg-white dark:bg-stone-900 rounded-2xl p-5 shadow-sm border border-stone-200 dark:border-stone-800">`
- List all 7 days of the current week (Mon-Sun)
- For each day: format using `format.dateTime(new Date(iso + 'T12:00:00'), { weekday: 'long', month: 'long', day: 'numeric' })` — IMPORTANT: append `T12:00:00` to avoid timezone day-boundary issues (see Pitfall 4 in research)
- Color code: delivery days in `text-green-700 dark:text-green-300`, non-delivery days in `text-stone-400 dark:text-stone-500`
- Add a subtle indicator for delivery days (e.g., a small green circle or checkmark before the date text)
- Highlight today's row: if `iso === todayISO`, add a subtle background highlight like `bg-stone-100 dark:bg-stone-800 rounded-lg` and append `t('today')` label
- Each day as a flex row: `<li className="flex items-center gap-3 py-2">`

**8. Visual design (VIS-02):**
- Warm color palette: stone/amber backgrounds, no sharp geometric shapes
- Rounded corners on cards (`rounded-2xl`)
- Generous padding (`p-5`, `py-12`)
- Subtle shadows (`shadow-sm`) and borders
- No harsh contrast jumps — use stone-50/100/200 gradient for depth

**9. Responsive (VIS-03):**
- Mobile-first: `max-w-sm` container, `px-4` padding
- Text sizes scale up at `sm:` breakpoint for question/answer
- Week view is full-width within the container
- Test at 375px viewport width mentally — all content must fit without horizontal scroll

**10. Dark mode (VIS-04):**
- Every color utility MUST have a `dark:` companion
- Background: `bg-stone-50 dark:bg-stone-950`
- Text: `text-stone-900 dark:text-stone-100` (base), `text-stone-500 dark:text-stone-400` (muted)
- Cards: `bg-white dark:bg-stone-900` with `border-stone-200 dark:border-stone-800`
- Delivery colors: green-600/green-400 for yes, red-600/red-400 for no
- Zero configuration needed — Tailwind v4 `dark:` uses `prefers-color-scheme` by default
